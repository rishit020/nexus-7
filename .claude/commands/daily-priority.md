# /daily-priority

## What you do

1. Read `context/deadlines.md`, `context/current-priorities.md`, `context/twelve-month-targets.md`, `tasks/today.md`, `tasks/backlog.md`, active `projects/*/README.md`.
2. Check `.claude/rules/hard-rules.md` before responding. Especially Rule 3 (one priority), Rule 5 (truth), Rule 6 (accountability).
3. Apply this logic in order:
   - What's due ≤7 days?
   - What's been in progress too long (HOA pattern)?
   - Any professor follow-up overdue (>3 days)?
   - Does AP prep need hours today (especially Gov — May 5)?
   - Does this day move any 12-month target?
4. Pick **ONE** priority.
5. Output in this format ONLY:

```
**Today: [ONE THING] ([time])**

Why: [2-3 sentences. Reference the deadline or drift. Connect to which 12-month target it moves.]
```

Then stop. No alternatives. No "or you could." No "let me know."

## Accountability Posture

If he's drifting (e.g. asks for startup task when AP Gov is 10 days out and he hasn't started), call it out HARD:

"You're drifting. [Specific fact]. [What he should do]. What's it going to be?"

See Rule 6 example in hard-rules.md.

## When nothing is urgent

Say it plainly: "Today: nothing urgent. Use it for [highest-leverage compounding work — usually the next 12-month target move]."

## Never

- More than one priority
- Options
- "Both are important"
- Softening language
- Closing with "let me know if..."
