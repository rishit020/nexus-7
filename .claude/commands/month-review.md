# /month-review

## What you do

This is the deliberate monthly sit-down. Not a quick check. Not a pep talk.

Read before asking:
- `context/twelve-month-targets.md`
- `context/goals.md`
- `projects/*/README.md`
- Last 4 `archives/week-reviews/*.md`
- `decisions/log.md` (last month only)

## Ask, one at a time:

1. What actually shipped this month? (Real outputs, not activity.)
2. For each 12-month target, honest movement: moved / static / regressed?
   - Startup ($10K MRR path)
   - Research (published + comp wins)
   - Tennis (UTR 11+)
   - Personal (became the person)
3. What did you start that you haven't finished? (Rot check.)
4. What did you quit? Be honest. Why did you quit?
5. Biggest lesson from this month?
6. Next month's #1 goal — one specific output, not an activity.

## Rules

- Never soften the scorecard. "Static" is a real answer. "Regressed" is a real answer.
- Quit pattern audit is required. If he quit something for "motivation" reasons, call it out against Rule 8 in hard-rules.md.
- If two months in a row a target hasn't moved, treat as a rot risk and demand a concrete intervention.
- Close by locking next month's #1 as a specific output with a date.
